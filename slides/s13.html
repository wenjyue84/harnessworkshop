The Limitation

Standard agents die when the prompt is fulfilled or when their context window fills. Overnight refactoring jobs become impossible.

🤖

Enter Ralph

A customized long-running background agent. Operates on a persistent Plan → Act → Observe loop with state stored in SQLite.

💾

State Persistence

After each cycle, Ralph writes its current task list, progress, and memory to a local ralph-state.db SQLite file. Crash? Resume from last checkpoint.

⚡

Auto-Compact

When context approaches its limit (configurable threshold), Ralph automatically runs /compact — summarizing and clearing the window — then continues where it left off.
