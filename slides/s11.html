Agents = Context Management

💡 Feynman Explanation

Forget the idea that an "agent" is an AI playing a role. An agent is a system that carefully decides what information gets put in front of the LLM at each step. The "role" is just one small piece of that information.

AK
"Context engineering is the delicate art and science of filling the context window with just the right information for the next step."
— Andrej Karpathy  ·  June 2025
TL
"The art of providing all the context for the task to be plausibly solvable by the LLM."
— Tobi Lütke (Shopify CEO)  ·  June 2025
HC
"Most of the time when an agent is not performing reliably the underlying cause is that the appropriate context, instructions and tools have not been communicated to the model."
— Harrison Chase (LangChain)  ·  June 2025
"Agents are typically just LLMs using tools based on environmental feedback in a loop." — Anthropic, "Building Effective Agents" (Dec 2024)

🔄 The Reframe

Designing a persona → Engineering what's in the system prompt
Role-playing an AI → Controlling what the LLM sees at each step
Prompt writing → Context architecture

What actually defines an agent:

System Prompt "You are a Prisma Coder. Use bun only..."
Memory (RAG) relevant code snippets from codebase
Tool Results file reads, test output, API responses
Conversation prior turns, observations, decisions
Current Task "Write auth middleware for /api/login"
