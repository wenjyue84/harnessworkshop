The Autonomous Agent Team

Instead of one general-purpose agent, deploy three hyper-specialized agents with clear adversarial roles. Each has a different context window, different constraints.

💻
Role: Builder

The Coder

Receives a scoped task from the Manager. Writes TypeScript logic strictly within the constraints of claude.md. Never tests. Never reviews. Just builds.

Context: only the task spec + codebase
💣
Role: Adversary

The Tester

Has one goal: break the Coder's work. Writes aggressive edge-case Playwright tests. Passes only when it cannot find a single failure mode.

Context: only the Coder's output
👁
Role: Judge

The Reviewer

Reads the git diff + Tester's execution logs. Either merges the PR or sends it back with specific, actionable change requests. No fluff.

Context: diff + test results only
