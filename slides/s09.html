Claude Code Hooks

Hooks are shell commands that fire at specific lifecycle events — turning your AI workspace into a self-maintaining system.

☀

on_start

The Morning Routine
Fires: Session Start
Runs the moment your AI workspace boots. Automates the daily setup ritual — so the AI starts each session with a clean, ready environment without being asked. Typical tasks: install deps, clear old logs, spin up dev server, check git status.
✓ bun install
✓ rm -rf .logs/old
✓ bun dev &
✓ git status
🔒

pre_commit

The Gatekeeper
Fires: Before Commit
Blocks the AI from committing broken code. Before any git commit proposal, this hook forces the AI to run your full test suite + linting. If tests fail, the commit is denied — the AI must fix the code first. This is your automated quality gate.
bun test
→ checking 42 tests...
✓ all tests pass
bun lint
✓ no lint errors
✨

post_generate

The Cleanup
Fires: After Code Gen
Runs immediately after the AI generates code. Auto-formats every file using Prettier to enforce Prisma standards — removing the need to ever argue about formatting in code reviews.
prettier --write .
✓ formatted: auth.ts
✓ formatted: api.ts
✓ 3 files updated
🔧

Hands-On: Add a pre_commit Hook

Exercise
1
Open your Claude Code settings

In your project root, find or create .claude/settings.json

# Claude Code settings location
.claude/
  settings.json   ← this file
2
Open the file in VS Code

Run: code .claude/settings.json

1
Add the hooks section
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "command": "bun test --passWithNoTests"
      }]
    }]
  }
}
1
Ask Claude to commit something broken

Type: "Commit this change" on a file with a failing test. Claude should run the hook, see the failure, and refuse to commit.

2
Fix the test and try again

Claude will only be able to commit once all tests pass. The harness is working.
