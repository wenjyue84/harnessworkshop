Context7 — Live Docs for Your LLM

Context7 is an MCP server by Upstash. Add use context7 to any prompt and it fetches the latest, version-specific documentation for your library — injecting it directly into Claude's context. No more hallucinated APIs.

📚
The Open Cookbook

Without Context7, asking Claude about a library is like asking someone to cook from memory — they'll get most of it right, but might use a deprecated method name or reference an API that changed in the latest version. Claude's training data has a cutoff; libraries keep changing.

Context7 hands Claude the actual cookbook, open to the right page, for the exact version you're using. Not 6-month-old training data — the live docs, right now.

ℹ
Context7 is an MCP server by Upstash — not a CLI tool, not a VS Code extension. It intercepts your prompt when it sees use context7, fetches docs for detected libraries, and injects them.

⚙️
Setup (2 minutes)
// ~/.claude/.mcp.json — add this server { "mcpServers": { "context7": { "command": "npx", "args": ["-y", "@upstash/context7-mcp"] } } }

No API key needed. No account. Just add the MCP and append use context7 to prompts involving external libraries.

💬
How to trigger it

Just append "use context7" anywhere in your prompt. Context7 detects libraries mentioned in your prompt, fetches the version-specific docs, and injects them before Claude responds.

# These all work: "Set up a Prisma schema with relations. use context7" "Write a Next.js Server Action for form submission. use context7" "How do I configure Drizzle ORM with PostgreSQL? use context7" "Create a Zod schema for user registration. use context7"

Especially valuable for: Next.js App Router TanStack Query v5 Prisma Drizzle ORM shadcn/ui

Claude Code — context7 in use
# Verify context7 is connected:
$ claude mcp list

> Write a Prisma query that finds users with at least 3 posts. use context7

context7: Fetching Prisma docs (v5.x)...
context7: Injecting 2,400 tokens of documentation

Using Prisma 5.x findMany with _count filter:
const users = await prisma.user.findMany({
where: { posts: { _count: { gte: 3 } } },
include: { _count: { select: { posts: true } } }
})
