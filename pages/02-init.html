/init Command

Running /init tells Claude to study your project and write its own CLAUDE.md — a contract that persists across every conversation. One command, permanent context.

The Self-Writing Onboarding Doc

Imagine hiring a new employee. Instead of you writing their onboarding document, you ask them to shadow your codebase for an hour and write their own handbook. That is /init — Claude reverse-engineers the instructions it needs to work well in your project.

Claude reads your package.json, README, existing code patterns, folder structure, and config files. Then it writes a CLAUDE.md tailored to your project — capturing the stack, conventions, and architectural decisions it can infer. Every future session loads this file automatically before the first message.

💡
Run /init once per project. After that, Claude carries your project context into every session automatically. Edit the generated CLAUDE.md to correct any wrong assumptions.

Hands-On Exercise

Exercise 02 — Bootstrap Your Project8 min
1
Open a terminal in any project folder
Use an existing project, or create one: mkdir my-app && cd my-app && git init
2
Start Claude Code
Run claude to launch Claude Code in that directory. You will see a prompt with your project path.
3
Run /init
Type /init and press Enter. Claude will start scanning your project. This takes 30-60 seconds depending on project size.
4
Wait for completion
Watch the output as Claude scans files and generates your CLAUDE.md. When it says "created!" the file is ready.
5
Review the generated CLAUDE.md
Open CLAUDE.md in your editor. Read what Claude wrote about your project. Look for any incorrect assumptions.
6
Edit incorrect assumptions
Fix anything Claude got wrong. Then start a new session and verify Claude loads the updated context.
⚠
Commit CLAUDE.md to git. This file should be version-controlled. It is documentation that helps Claude and new team members understand your project.

Key Takeaways

📄
CLAUDE.md persists across all sessions in that project. Claude reads it at the start of every conversation automatically — no manual loading needed.
🔏
/init is a one-time command per project. Run it once, edit the output, and Claude carries that context permanently into future sessions.
✏
Always review and edit the generated file. Claude makes good inferences but also makes assumptions. Correct them before your next session for the best results.
