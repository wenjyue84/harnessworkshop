Ralph

Long-Running Background Agents

Ralph is a pattern for long-running Claude Code agents. While you sleep, Ralph plans tasks, executes them step by step, uses /compact to manage context, saves state to a file, and picks up where it left off. It is the foundation of autonomous multi-hour work sessions.

📋

The Overnight Assistant Metaphor

Most AI interactions are like asking for directions — you get an answer and walk away. Ralph is like hiring a trusted assistant who works overnight: they have a to-do list, they work through it methodically, they note what is done and what is blocked, they /compact when running low on context, and in the morning they leave you a status report. For Prisma, Ralph builds the entire IoT dashboard overnight, migrates the payroll database while clients are offline, or deploys three client chatbots in sequence while the team sleeps.

Ralph’s Operating Instructions

ralph.md
# ralph.md — Prisma IoT Dashboard Build Instructions
You are Ralph, a long-running autonomous agent for Prisma Technology Solution.

## Your overnight task loop:
1. Read prisma-iot-tasks.md
2. Pick the next incomplete task
3. Execute using Bash, Read, Write tools
4. Mark it done in prisma-iot-tasks.md
5. If context > 80%: run /compact
6. Repeat until all dashboard tasks done
7. Write final report to RALPH-REPORT.md

# Tasks: React dashboard, MQTT backend, Telegram alerts

Exercise: Run Ralph

Build a Ralph Loop Agent

1

Create ralph-tasks.md for a Prisma sprint

Write 5 real tasks: 1. Audit prisma-payroll EPF calculation, 2. Add Telegram alert to IoT monitor, 3. Write unit tests for SOCSO deductions, 4. Update chatbot FAQ for Pelangi Hostel, 5. Generate client report for Federation Oil.

2

Create ralph.md with Prisma context

Include: You are Ralph, Prisma Technology's autonomous dev agent. Read ralph-tasks.md, pick the next task, execute it fully, save progress to ralph-state.json, repeat.

3

Launch Ralph

claude --dangerously-skip-permissions "" in a Prisma project directory.

4

Watch Ralph work

Ralph processes all 5 tasks autonomously. This is a realistic Prisma sprint — the kind of work that would take a developer half a day manually.

5

Review outputs

Check ralph-log.txt and RALPH-REPORT.md. Verify payroll calculations against LHDN tables, test Telegram messages, read the Federation Oil report.

6

Advanced: /compact trigger

Add context monitoring: if context exceeds 80%, trigger /compact so Ralph can keep running without losing context across your IoT codebase.

Key Takeaways

🔄Ralph = autonomous loop agent with Plan-Act-Observe-Save cycle. Builds Prisma's IoT dashboard overnight.
📋Needs a Prisma task list (prisma-iot-tasks.md), operating instructions, and a state file.
🧹/compact is essential for long payroll migration runs — prevents context overflow mid-task.
🎯Works best with discrete subtasks: MQTT backend, React dashboard, Telegram alert deployment in sequence.
🔍Always review the RALPH-REPORT.md and test payroll figures after a Ralph run before delivery to clients.
