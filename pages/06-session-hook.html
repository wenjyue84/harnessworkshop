Session Start Hook

Claude Code Hooks let you run shell commands at specific lifecycle events. The SessionStart hook fires every time you open Claude Code — before any conversation. Use it to auto-load context, check git status, or inject the current date.

The Surgeon Pre-Flight Checklist

Before any surgeon operates, the team runs a pre-flight checklist: patient verified, tools sterile, team briefed. The session start hook is your Prisma development pre-flight checklist — run it every time, without thinking about it.

Without a SessionStart hook, Claude opens into a blank context. With one, Claude enters every session already knowing today's date, the current git branch, the active Prisma client project, which deployment environment is active, and the MQTT broker status. You configure it once; it runs every time you open Claude in any project.

💡
Hooks are defined in ~/.claude/settings.json. The output from hook commands is visible in the session context. Claude reads it before your first message.

Hook Configuration

~/.claude/settings.json
// ~/.claude/settings.json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "echo Session start: $(date '+%d/%m/%Y %H:%M MYT'); echo Active project: $(cat current-project.json 2>/dev/null | node -p 'JSON.parse(require(\"fs\").readFileSync(\"/dev/stdin\",\"utf8\")).client' 2>/dev/null || echo unknown); git status --short"
          }
        ]
      }
    ]
  }
}

Hands-On Exercise

Exercise 06 — Your First Session Hook10 min
1
Open settings.json
Open ~/.claude/settings.json in your editor. Create it if it does not exist yet.
2
Find or create the hooks section
Add a top-level "hooks" key if it is missing. The structure is an object with event names as keys.
3
Add a SessionStart hook
Copy the JSON from the terminal above. The command runs date and git status --short at session start.
4
Save and restart Claude Code
Save settings.json, then open a new Claude Code session. You should see hook output at the top of the session.
5
Observe the output
The date and git status from your hook will appear as injected context. Ask Claude "what branch am I on?" — it should know from the hook output.
6
Upgrade your hook with project context
Create a current-project.json file in your project root with fields like { "client": "Federation Oil", "env": "staging", "stack": ["MQTT", "Supabase"] }. Read it in the hook so Claude always knows which client's system it is working on.

Key Takeaways

△
SessionStart hooks inject context before conversations begin. Claude reads hook output before your first message. Use this for date, git status, and health checks.
🐠
Hooks run as shell commands. Any valid shell command works. Chain multiple commands with semicolons or write a shell script.
⚡
Set it once, benefit every session. A good SessionStart hook is invisible until you need it. It runs silently and loads context you would otherwise forget to provide.
