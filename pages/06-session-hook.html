Session Start Hook

The SessionStart hook fires every time Claude Code opens — before any conversation. Use it to automatically load context: git status, today's date, health checks, reminders.

🏥
The Pre-Flight Checklist

Before any surgery, the team runs a checklist: patient verified, instruments counted, team briefed. No surgeon skips this. It takes 2 minutes and prevents costly mistakes.

A SessionStart hook is your AI's pre-flight checklist. Every time Claude Code opens, it automatically runs your commands, injects the output into context, and Claude starts the session already informed. Zero effort, every session.

How to configure it

Hooks live in ~/.claude/settings.json. The SessionStart key takes an array of hook objects, each with a command field.

// ~/.claude/settings.json { "hooks": { "SessionStart": [ { "matcher": "", "hooks": [ { "type": "command", "command": "echo '=== Session Start ===' && date && git status --short 2>/dev/null || echo 'Not a git repo'" } ] } ] } }

💡
Useful hook commands
DATE & CONTEXT
echo "Today: $(date +%Y-%m-%d)"
GIT STATUS
git log --oneline -5 && git status --short
LOAD NOTES
cat ~/daily-notes/$(date +%Y-%m-%d).md 2>/dev/null
HEALTH CHECK
node --version && npm --version

Hands-On Exercise

01

Open settings.json

Open ~/.claude/settings.json in your editor. If it doesn't exist, create it with {}.

02

Add the SessionStart hook

Add the JSON structure shown above. Start with a simple command: "echo 'Today is $(date +%Y-%m-%d)'; git status --short 2>/dev/null"

03

Restart Claude Code

Exit and reopen Claude Code in a git repository. You should see the hook output appear before your first message.

04

Enhance it

Add more useful commands: show the last 3 git commits, load a project-specific status file, or print any TODOs from your notes.
