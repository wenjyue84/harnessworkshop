Module 19 Multi-Agent 15 min

Agent Team

Coder + Tester + Reviewer

A single agent makes mistakes without accountability. For Prisma's payroll system, Malaysian tax rates must be exact. A Coder, Tester, and Reviewer working in adversarial roles catches the edge cases a single agent would miss — overtime pay, age 60+ EPF rates, SOCSO ceiling breaches.

👥

The Code Review Metaphor

One developer reviewing their own EPF calculation misses 60% of edge cases — they check what they intended, not what the code actually computes at salary boundaries. Add a Tester agent and the bug catch rate triples. Add a Reviewer and it triples again. For Prisma's payroll, a wrong PCB deduction means client audit risk — adversarial agent teams are the quality gate.

Exercise: Sequential Role Agent

Three-Role Agent Exercise

1

Pick Prisma's payroll function

Use calculateEPF(grossSalary, age) — a function that must comply with KWSP 2025 contribution tables. Wrong output = client audit failure.

2

Coder role

Ask Claude: Act as Coder and implement calculateEPF(grossSalary, age) using KWSP 2025 rates. Employee 11%, employer 13% if salary under RM5,000.

3

Tester role

Ask Claude: Now act as Tester. Try to break calculateEPF with: salary exactly RM5,000, salary RM5,001, age 60+, zero salary, negative salary, RM0.01. Report each failure.

4

Reviewer role

Ask Claude: Act as Reviewer. Critique the EPF implementation for LHDN compliance, missing rate bands, incorrect employer percentage, and absent audit logging.

5

Coder addresses all feedback

Have the Coder fix every failure the Tester found and every concern the Reviewer raised. This loop continues until the payslip is LHDN-compliant.

6

Observe quality improvement

Compare the final calculateEPF vs a single-agent attempt. The three-role team catches the age 60+ rate edge case and salary ceiling that a single agent misses.
