Global CLAUDE.md

A global CLAUDE.md lives at ~/.claude/CLAUDE.md and loads in every Claude Code session — across ALL projects. It is your universal AI constitution.

The Laminated Card Every Employee Carries

Imagine you could give every Prisma developer — across every client project — a laminated card with core rules. "Always use Malaysian date format DD/MM/YYYY. Never hardcode IoT device keys or Telegram bot tokens. All statutory deductions must reference LHDN rates." That is global CLAUDE.md. It loads before any project instructions.

The global file defines how Claude behaves as your AI — your communication style preferences, your tooling defaults, your deletion policy. Every Prisma project inherits these rules automatically — whether you are working on the Federation Oil IoT dashboard or the Pelangi Hostel chatbot. The project-level CLAUDE.md can then override specific rules for that context.

💡
Think of it as personality, not policy. Global CLAUDE.md shapes how Claude approaches all work. Project CLAUDE.md shapes what Claude knows about a specific codebase.

A Sample Global CLAUDE.md

~/.claude/CLAUDE.md
# Global CLAUDE.md location: ~/.claude/CLAUDE.md
$ cat ~/.claude/CLAUDE.md

# Prisma Technology Solution — Global Rules

## Date & Locale
- Use Malaysian date format: DD/MM/YYYY (e.g. 24/02/2026).
- Currency in MYR. Statutory rates follow LHDN / EPF / SOCSO guidelines.

## Credentials & Security
- Never hardcode Telegram bot tokens, MQTT broker passwords, or IoT device keys.
- All secrets go in .env files. Never commit .env to git.
- Always read a file before editing it.

## File Safety
- Never permanently delete files. Archive to _archive/ instead.

## Naming
- Use kebab-case for all file and folder names.
- Bahasa Melayu comments are acceptable alongside English in shared modules.

## Communication Style
- Be concise and direct. No cheerleading or filler phrases.
- No emojis in code comments or technical responses.

Exercise 03 — Write Your Global Rules10 min
1
Find the global CLAUDE.md path
In Claude Code, run /config to see the config path. The global CLAUDE.md lives at ~/.claude/CLAUDE.md (or the path shown).
2
Create or open the file
Run: code ~/.claude/CLAUDE.md or open it in any text editor.
3
Add a credentials safety rule
Example: - Never hardcode Telegram bot tokens, MQTT passwords, or IoT device keys. All secrets must be in .env.
4
Add your deletion policy and date format
Example: - Never permanently delete files — archive to _archive/ instead. and - Use DD/MM/YYYY for all Malaysian date outputs.
5
Start a new Claude Code session
Open Claude Code in any Prisma project. Verify it loaded your global rules by asking: "What date format should you use for output?" and "Where do secrets go?"
