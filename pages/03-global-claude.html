Global Claude.md

A global CLAUDE.md loads in every Claude Code session, across all your projects. It's your universal AI constitution — rules that apply everywhere, before any project-specific instructions.

🏛️
The Laminated Card

Imagine you could give every employee across every company you ever work with a laminated card: "I always use uv for Python. I never permanently delete files. I write tests first." No matter what project, no matter who you're working with — those rules always apply.

That's global CLAUDE.md. It loads before any project instructions and sets your baseline defaults across your entire AI workflow.

📐 Instruction Hierarchy

Instructions are merged in this order. Higher layers override lower ones when there's a conflict.

Base Layer always present
Claude's built-in defaults and safety behaviors
Global CLAUDE.md ~/.claude/CLAUDE.md
Your personal rules — applies to every project on this machine
Project CLAUDE.md ./CLAUDE.md
Project-specific rules — overrides global for this repo
💡
When there's a conflict, the more specific layer wins. A project rule "always use pip" overrides a global rule "always use uv" for that project.
📝
Example Global CLAUDE.md
# ~/.claude/CLAUDE.md — Global Rules (applies everywhere) ## Critical Rules - Use `uv` for ALL Python operations. Never use pip directly. - Read a file before editing it. Never edit without reading first. - Archive instead of delete. Move to _archive/ if uncertain. ## File Naming - Documents: YYYY-MM-DD_name.ext prefix - Code: kebab-case for files, camelCase for variables ## Communication Style - Be concise. No cheerleading or excessive affirmations. - No emojis unless explicitly requested. - When you don't know something, say so directly. ## Platform - Windows 11. Use PowerShell-compatible syntax. - Default region: ap-southeast-1

Hands-On Exercise

01

Find your global CLAUDE.md

Open Claude Code and run /config to see the exact path. On Windows, it's typically C:\Users\[you]\.claude\CLAUDE.md.

02

Write your critical rules

Open the file and write at least 3 rules you want applied in every session. Think about: your preferred tools (Python env, package manager), your deletion policy, your communication style preferences.

03

Add platform context

Tell Claude your OS, shell, and any special environment facts. This prevents Claude from suggesting Linux commands when you're on Windows, or using the wrong path separators.

04

Test it in a new session

Exit Claude Code, restart it, and verify: ask "What are your rules?" — it should reference your global CLAUDE.md content.
