Codex CLI Installation

terminal
# Install Codex CLI:
$ npm install -g @openai/codex

# Configure with API key:
$ export OPENAI_API_KEY=sk-...

# Run in Prisma's IoT dashboard project:
$ codex "Add TypeScript types to all sensor data interfaces in src/types/"

# Yolo mode (like dangerously-skip-permissions):
$ codex --full-auto "Add error handling to all MQTT API calls in src/mqtt/"
Running autonomously across all MQTT handler files...

Exercise: Set Up Codex + Cursor

Multi-Tool AI Setup

1

Install Cursor

Download from cursor.sh and install.

2

Install Codex CLI

npm install -g @openai/codex

3

Add API key

Add your OpenAI API key to your environment variables.

4

Open Prisma's IoT dashboard in Cursor

Open prisma-iot-counter/src/ in the Cursor IDE. See how Cursor understands the React component tree immediately.

5

Try Cmd+K for TypeScript IoT types

Place cursor in a sensor data handler function and press Cmd+K: "Add TypeScript interface for MQTT sensor payload with machineId, count, timestamp, status."

6

Try Codex CLI on IoT components

codex "Add PropTypes to all React components in src/components/ of the Prisma IoT dashboard" in the Cursor terminal.

7

Compare to Claude Code

Use Claude Code for the same task. Note: Cursor excels at inline suggestions within the file you are editing. Claude Code excels at multi-file autonomous changes across the whole IoT codebase.

Key Takeaways

📝Cursor excels at inline TypeScript completions for Prisma's IoT sensor interfaces and React dashboard components.
➕Both complement Claude Code, not replace it. Use each for what it does best.
🧵Use Claude Code for overnight IoT builds, payroll scaffolding, and hook-driven autonomous workflows.
🖥️Use Cursor/Codex when actively editing React dashboard components or typing MQTT handler logic inline.
