Project Claude.md

Every project has a different stack, team, and set of rules. Project-level CLAUDE.md lives in the repo root and gives Claude exactly the context it needs to work in this specific codebase.

🏢
The Workplace Handbook

Your global CLAUDE.md describes how you generally work. But every workplace has its own culture, tools, and forbidden practices. A React shop has different rules than a Django shop. A startup has different deployment rules than an enterprise.

Project CLAUDE.md is the employee handbook for this specific team. New AI session? It reads the handbook first. Same Claude, different project? Different handbook. No context lost between sessions.

# CLAUDE.md — my-nextjs-app ## Stack - Next.js 14 (App Router, Server Components) - TypeScript strict, Tailwind CSS, shadcn/ui - Database: Prisma + Neon PostgreSQL - Auth: NextAuth.js v5 ## Conventions - Server Components by default; add "use client" only when needed - Form validation: Zod schemas in lib/validations/ - API routes: app/api/ with route.ts pattern ## Commands - Dev: npm run dev (localhost:3000) - DB migrate: npx prisma migrate dev - Type check: tsc --noEmit ## Do Not Modify - src/components/ui/ — shadcn auto-generated - .env.production — never touch
# CLAUDE.md — my-django-api ## Stack - Django 4.2 + DRF 3.14 - Python 3.12 — ALWAYS use uv, never pip - PostgreSQL + Redis (celery tasks) - JWT auth via djangorestframework-simplejwt ## Conventions - Each app has: models.py, serializers.py, views.py, urls.py - Use class-based views (ViewSet pattern) - All API responses: {"data": ..., "meta": ...} format ## Commands - Dev: uv run python manage.py runserver - Test: uv run pytest -x - Migrate: uv run python manage.py migrate ## Critical - Never store credentials in settings.py — use env vars - Always write migrations — never edit existing ones
# CLAUDE.md — my-mobile-app ## Stack - React Native 0.74 + Expo SDK 51 - TypeScript, NativeWind (Tailwind for RN) - State: Zustand + React Query - Navigation: Expo Router (file-based) ## Platform Rules - Target: iOS 16+ and Android 13+ - Test on both platforms before committing - Use Platform.select() for platform-specific code ## Commands - Dev: npx expo start - iOS: npx expo run:ios - Android: npx expo run:android ## Do Not - Never use web-only APIs (window, document, localStorage) - Don't use px units — use dp/rem only

Hands-On Exercise

01

Create CLAUDE.md in your project root

Navigate to any project. Create a file called CLAUDE.md in the root directory (same level as package.json or main config file).

02

Write the 5 essential sections

Add: Stack (what you're using), Conventions (patterns to follow), Commands (how to run things), Architecture (key structure notes), Do Not (things to never do).

03

Test it works

Run claude in that folder, then type /status. Confirm your CLAUDE.md was loaded.

04

Commit to git

Run git add CLAUDE.md && git commit -m "feat: add CLAUDE.md for AI context". Your team now shares consistent AI context.

💡
Add .claude/ to .gitignore if you don't want personal session data committed — but CLAUDE.md itself should be committed.
