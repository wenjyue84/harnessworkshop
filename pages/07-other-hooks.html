Other Hooks

Beyond SessionStart, Claude Code exposes hooks for every significant event: PreToolUse, PostToolUse, Notification, and Stop. These let you build audit trails, safety checks, auto-formatters, and notification systems.

The Professional Kitchen Station Model

A professional kitchen has stations: prep (before cooking), quality check (after each dish), expeditor (before it leaves). Claude Code hooks are those stations — you intercept actions at every stage and add your own logic. At Prisma, this means blocking accidental production deployments, logging every database migration for audit, and notifying the team on Telegram when a deployment completes.

PreToolUse hooks can inspect and block. PostToolUse hooks can log, notify, or trigger side effects. The Notification hook fires when Claude needs your attention. The Stop hook fires when Claude finishes a task. Together, they give you a complete audit and control system around every AI action on client systems.

⚠
PreToolUse can block operations. If your hook exits with a non-zero code, the tool call is cancelled. Use this for safety checks on destructive operations like file deletion.

PostToolUse Audit Log Hook

~/.claude/settings.json
// Log every DB migration and notify team on Telegram when deployment completes
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit|Bash",
        "hooks": [{
          "type": "command",
          "command": "echo \"[$(date '+%d/%m/%Y %H:%M')] Modified: $CLAUDE_TOOL_INPUT\" >> ~/prisma-audit.log"
        }]
      }
    ],
    "Stop": [{
      "matcher": "deploy|migration",
      "hooks": [{
        "type": "command",
        "command": "curl -s -X POST $TELEGRAM_WEBHOOK_URL -d '{\"text\":\"Prisma deploy complete on $(hostname)\"}' > /dev/null"
      }]
    }]
  }
}

# Result: every file edit is audited; deployments notify the Prisma team
$ cat ~/prisma-audit.log
[24/02/2026 09:45] Modified: src/mqtt/broker.js
[24/02/2026 09:47] Modified: src/sensors/water-level.js
[24/02/2026 10:02] Modified: supabase/migrations/20260224_add_shift_column.sql

Hands-On Exercise

Exercise 07 — Build an Audit System12 min
1
Create a PostToolUse audit hook
Add the JSON from the terminal above to your settings.json. The matcher Write|Edit|Bash fires on file modifications and shell commands. Logs go to ~/prisma-audit.log with Malaysian date format.
2
Test it with a real edit
Open Claude Code in iot-monitor and ask Claude to edit a sensor file. Afterward, check ~/prisma-audit.log — you should see the timestamp and file name logged.
3
Create a PreToolUse safety hook to block production IoT deploys
Add a hook that blocks Bash commands containing deploy or pm2 restart when the environment is production. Exit code 1 cancels the command before it runs on live IoT devices.
4
Advanced: Telegram notification on deployment complete
Add a Stop hook that sends a message to your Prisma team Telegram group using curl and your bot token stored in $TELEGRAM_WEBHOOK_URL. Never hardcode the token — read it from environment variables.
ℹ
Environment variables in hooks: $CLAUDE_TOOL_INPUT contains the tool input JSON. $CLAUDE_TOOL_NAME contains the tool name (Write, Edit, Bash, etc.).

Key Takeaways

🔐
Hooks give you control at every lifecycle stage. Before, during, and after every tool call. You are never flying blind on what Claude is doing.
🚫
PreToolUse can block dangerous operations. Exit with a non-zero code to cancel the tool call before it executes. Perfect for deletion guards and access control.
📄
PostToolUse is perfect for logging and audit trails. Every file write, every bash command, every edit — all can be silently logged to a file for later review.
