Other Hooks

Beyond SessionStart, Claude Code exposes hooks at every lifecycle stage: before tool calls, after tool calls, on notification, and on stop. Build audit trails, safety checks, auto-formatters, and alerts.

🍳
The Kitchen Stations

A professional kitchen has quality control stations: the prep station checks ingredients before cooking, the pass checks plates before they leave. If anything is wrong, it gets stopped before reaching the customer.

Claude Code hooks are those stations. PreToolUse can inspect and block actions before they happen. PostToolUse can log, notify, or trigger follow-up actions after. You're adding control points to the AI's workflow.

📋 All Hook Events
Event When it fires Can block? Best for
SessionStart Session opens, before any message No Injecting context, date, git status
PreToolUse Before Claude calls any tool YES Safety checks, confirmations, validation
PostToolUse After a tool call completes No Audit logs, notifications, auto-formatting
Notification Claude sends a notification No Desktop alerts, Slack/webhook forwarding
Stop Claude finishes responding No Summary writes, CI triggers, cleanup
📝
PostToolUse: Audit Log

Every file write gets logged automatically. No manual tracking needed.

// ~/.claude/settings.json { "hooks": { "PostToolUse": [{ "matcher": "Write|Edit|Bash", "hooks": [{ "type": "command", "command": "echo \"$(date +%Y-%m-%dT%H:%M:%S) TOOL=$CLAUDE_TOOL_NAME\" >> ~/claude-audit.log" }] }] } }
🛡️
PreToolUse: Safety Block

Exit with code 2 from a PreToolUse hook to block the tool call. Claude receives your error message and adjusts.

// Block deletion of .env files { "PreToolUse": [{ "matcher": "Bash", "hooks": [{ "type": "command", "command": "bash -c 'if echo \"$CLAUDE_TOOL_INPUT\" | grep -q \"rm.*\\.env\"; then echo \"BLOCKED: cannot delete .env files\" >&2; exit 2; fi'" }] }] }
⚠
Exit code 2 = block. Any other non-zero exit = error (hook failed but tool still runs). Exit 0 = success, tool proceeds. The error message in stderr is shown to Claude.
bash — hook in action
# Claude tries to delete a .env file
> running: rm .env.local
Hook blocked: cannot delete .env files
PreToolUse hook rejected the action.

# Claude receives the block and adjusts:
I won't delete .env.local. Should I move it to .env.backup instead?
