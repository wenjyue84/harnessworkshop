Agent / Subagent

Context Management Hierarchy

When a task is too large for one context window, you split it. The main agent handles the plan and coordination; subagents handle specific subtasks in their own isolated contexts. The key insight: an agent is not a role — it is a system that engineers exactly what tokens sit in the context window.

💼

The Project Manager Metaphor

A project manager does not personally write every line of code, conduct every interview, and draft every email. They spawn specialists: the coder, the researcher, the writer. Each specialist has their own workspace (context window) and reports back. The PM synthesizes the results. That is the agent/subagent pattern.

Key Takeaways

💡Agents = context engineering, not roleplay. Engineering is what tokens sit in RAM.
✂️Split large tasks into subagents when context is the bottleneck.
🎯Each subagent gets only what it needs — focused context.
🧩Main agent synthesizes. This scales beyond a single context window.
