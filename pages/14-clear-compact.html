/clear and /compact

Context Window Management — knowing when to compress vs when to start fresh.

Claude context window is finite. /clear resets it completely — fresh start, clean slate. /compact summarises and compresses the conversation — keeps the important parts but removes the bulk. For Prisma, this matters when switching between client projects (Federation Oil IoT vs Pelangi chatbot) or after a long payroll system build session.

💡
The Mental Model
Think of the context window as your desk. After hours of debugging Prisma's IoT sensor code, the desk is covered in MQTT logs, sensor readings, and half-finished notes. /compact is like taking a photo of the important stuff and clearing the clutter. /clear is like pushing everything off completely when you switch from the IoT project to the Pelangi chatbot.

📊 /compact vs /clear

📦 /compact

/compact
  • Summarises the conversation into a dense summary
  • Keeps payroll logic decisions and file names
  • Removes verbose EPF calculation debug back-and-forth
  • Context drops to ~15-25% after compaction
  • Task continuity is maintained
Use when: mid-payroll-build, context filling but task continues

⏬ /clear

/clear
  • Erases the entire conversation history
  • Claude has no memory of payroll work done
  • Fresh context — full 200K tokens available
  • You must re-explain Pelangi chatbot context if needed
  • Fastest way to switch between Prisma client projects
Use when: switching from Federation Oil IoT to Pelangi chatbot

🏋 Exercise

Hands-On: Master Context Management Across Prisma Projects
1
Build context on one project
Start a session working on Prisma's payroll system. Ask Claude to generate EPF, SOCSO, and EIS calculation functions. Do a few rounds of iteration with edge cases.
2
Check context usage
Ask Claude: How full is your context window right now? Note the percentage. Ask what Claude remembers from the payroll tax logic built so far.
3
Run /compact
Type /compact and observe what Claude includes in its summary. Verify it remembered the EPF/PCB/SOCSO decisions but shed the verbose debug dialogue.
4
Continue the payroll task
Keep working. Ask Claude to add CP22 form generation. Verify it still has full context of the tax calculations — the compact summary is working.
5
Switch projects with /clear
When the payroll task is done, type /clear and start fresh on the Pelangi Hostel chatbot project. Notice the clean slate — no leftover payroll context.
6
Compare the experience
Reflect: which command made sense when? /compact mid-payroll-build kept momentum. /clear at project switch prevented context contamination between clients.
📚
Pro Tip: Compact Early on Complex Payroll Sessions
Do not wait until the context is 90% full. Compact at 60-70% when the payroll logic summary will still be accurate and coherent. Late compaction on a complex Malaysian tax calculation session risks losing important PCB rate nuance because the summary has too much to squeeze.

🎯 Key Takeaways

What to Remember

💾
Context window = limited RAM. Every token you add displaces something. Manage it actively across Prisma's projects.
📦
/compact = lossless compression. Keeps EPF/SOCSO/PCB decisions and code references, removes dialogue bulk.
⏬
/clear = full reset. Use when switching between client projects: Federation Oil IoT to Pelangi chatbot.
🕑
Compact at 60-70%, not 90%. Earlier compaction produces better summaries of complex tax logic.
📊
Watch the token count. Ask Claude how full its context is whenever a payroll debug session feels sluggish.
