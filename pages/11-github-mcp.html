GitHub MCP

The GitHub MCP server connects Claude directly to the GitHub API. Read issues, create PRs, review code, search repositories — all without leaving the terminal or manually switching to the browser.

🔄
The Context Switch Tax

Normal GitHub workflow looks like this: read the issue in the browser switch, open your editor switch, write the code, open the terminal switch, push, open the browser again switch, create the PR, write the description, request a review switch.

Each context switch costs 20+ minutes of refocusing. GitHub MCP eliminates all the switches. Claude reads the issue, writes the code, creates the branch, opens the PR, and writes the description — in one continuous flow inside your terminal.

🐙 What Claude can do with GitHub MCP
📋

Read & Create Issues

List open issues, read issue details, create new issues, add comments, update labels and assignees

🔀

Pull Request Workflow

Create branches, list PRs, read PR diffs, create pull requests with descriptions, request reviewers, merge PRs

🔍

Search & Explore

Search code across repositories, find issues by keyword, search users, browse file contents

📊

Reviews & Status

Get PR review status, read review comments, check CI/CD status, view commit history

Hands-On Exercise

01

Create a Personal Access Token

Go to github.com → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token. Select scopes: repo, issues, pull_requests.

02

Add to ~/.claude/.mcp.json

Create or edit ~/.claude/.mcp.json with the configuration above. Replace ghp_your_token_here with your PAT.

03

Restart and verify

Restart Claude Code and run claude mcp list — you should see github in the list.

04

Test it

Ask: "List the last 5 issues in [your-org/your-repo]" then "Create a test issue titled 'GitHub MCP test'"
